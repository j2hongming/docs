    Tennis

    tennis test case 或是影片記得提前看，最好可以知道整個 tennis 的開發順序跟內容怎麼做 課前影片觀看重點不是按了什麼鍵，而是 tennis 的需求、測試案例，以及 TDD 的開發方式

    • PyCharm - TDD tennis v2
    • 活動介紹中的影片

    從0:0(love all)開始, 建立Tennis物件然後移動到另一個py檔 預期Tennis物件有一個負責計算分數的score method, 初始版本只有簡單地回傳love all 驗測的函式抽出為score_should_be

    接著開始測試1:0(fifteen love), Tennis物件加上能夠增加first player的score, 並在score method增加判斷邏輯並回傳fifteen love

    測試2:0(thirty love), 在Tennis物件score method增加判斷邏輯並回傳thirty love, 歸納後發現重複性, 可以用dict資料結構紀錄分數(1, 2)和呈現(fifty, thirty)的關係, 開始整理score method的邏輯, 告一段落後, 開始整理測試thirty love的重複邏輯, 接著測試3:0(forty love)

    測試0:1(love fifteen), 開始引入second player, Tennis物件加上能夠增加second player的score, 開始整理score method的邏輯, 測試0:2(love thirty), 整理score method的邏輯, 發現dict資料結構可以加上love=0的關係

    測試1:1(fifteen all), 修改score method的邏輯, 測試2:2(thirty all), 發現可以整理score method, 更加語義化, 在這個時候已經具備高可讀性

    測試3:3(deuce), 修改score method的邏輯在分數大於等於3時, return deuce, 測試4:3(first player adv), 修改score method的邏輯, 測試3:4(second player adv)

    測試win 最後整理score method, 不同分狀況下, 以game point區分, 同分狀況下, 以deuce區分

    :::warning 1. 為何是從0:0(love all)開始測試, 若我從1:0(fifteen love)開始會發生甚麼事 2. 1:2, 1:3, 2:1, 3:1好像沒測試 :::

    :::success - 確保Internal邏輯的修改(score method)不會影響到這一個Tennis版本的External Spec行為(test case) => validation by passing test :::

    vim

    :::success 最終目標, 練習用vim控制整個IDE :::

    mode

    normal mode: 移動, 選取, 下指令 insert mode: 編輯 visual mode: 選取

    normal => insert: i or a, I or A insert => normal: Esc(可以用其他key取代, 物理距離會比較順暢) normal => visual: v or V

    相對行號: set rnu 絕對行號: set rnu! => 轉為hot key比較快

    :::info vim世界, shift代表往前或選取的含意 :::

    移動

    :::success 最終目標, 練習在畫面上哪一個位置就要移動到那邊 ::: 字元層級(char) 方向: hjkl :::info 在同行左右移動會較少用到h or l (因為有wbe和ft) :::

    字(word層級) wbe WBE: 接在一起都算一個字, 遇到空白和斷行算下一個字

    行層級(line) 原生vim: ^ $ 0 :::info 可以用vim設定檔取代, 目的是解決距離太遠不好按的議題 hh ll :::

    段落 { } 成對符號 % :::info 比較少用 ( 和 ) :::

    視窗 HLM

    檔案 gg G :::info 行號+G: 跳到第幾行 （幾乎不用，因為相對行號更快）

    :::

    Scroll (keep mode) 常用 zt: current line, top zz: current line, middle

    搜尋 (keep mode) 單行搜尋某個char ft ;(repeat) ,(reverse, 不小心按太多次;)

    特殊鍵

    • ideaVim
      • Shift + 上/下：pageUp/pageDown
      • z右 or z左：水平捲軸往右/左，前面可加數字
      • ze or zs: 游標的位置當水平捲軸的結尾或開頭
    • vim 內建好用 shortcut
      • u: 復原（類似 ctrl+Z, 但認的是 vim 的動作）
      • number, Ctrl+A: 該行往右找，第一個數字累加 number
      • number, Ctrl+X: 該行往右找，第一個數字減去 number
    • daw (等同自訂組合鍵zb)
    • di"
      • 刪除雙引號內的內容
      • 游標不需要停在雙引號之內
    • di}
      • di}可用 diB 代替，而 di) 可用 dib 代替
      • 刪除目前所處的 {} block 內容
      • 同理可用在 () 或 [] 或 {}
    • dit
      • dit 可刪除 框起來內容
      • dat 可連 一併刪除
